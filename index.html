  2. Introduction

    1. Description

      SetMap is a simple to use and yet powerful jQuery plugin which helps your users to select a location on a map with either typing or dragging.

      It uses Google Maps and jQuery 1.6.4. It is strictly object-oriented and follows the implementation of the jQuery UI widgets.

      Thus, it allows you to create multiple instances with just few lines of javascript and to use previously created maps with the same jQuery objects.

      The plugin supports Geocoding, Reversed Geocoding and HTML5 Geolocation.

    2. Features

      • Embed multiple maps with a single line of jQuery code
      • Very easy to use and set only the options you need
      • Set map dimensions with pure CSS
      • Get coordinates and full address when the user drags the marker around the map
      • Uses HTML5 Geolocation to set the map to the user's current location./li>
      • Change the map location dynamically with coordinates or an address
      • Control the map zoom controls, map type, marker animation and more
      • Enables not draggable maps or static maps for showing off your venue or business
      • Add marker title to a location
      • Demos with great application of the plugin included
      • Thorough documentation with copy-paste examples
      • The source code is compliant with JSLint.
      • Follows the jQuery and jQuery UI design principles and best practices
      • Just 1.56kb gzipped

  3. Requirements

    • jQuery 1.4+

      The plugin can work with jQuery 1.4.0 or newer, but I recommend using the latest stable release 1.6.4 or 1.7 RC 2.

      The demos contain the best way to load jQuery following the recommendation from HTML5 Boilerplate.

      First you try to load it from the Google CDN and after that if jQuery is not laoded you load a local file.

    • Google Maps Javascript API v3

      You can use the Google Maps API for Javascript with adding a single javascript file.

      You can refer to the loading of the library in the demos. There are few parameters to be set in the URL of the script.

      Only the sensor parameter is required. It should be set to true only if you are using a device with GPS hardware.

      Other important parameters are region and language. The defaults are region=us and language=en.

      Note that the special features for Geocoding and Reversed Geocoding can be used only if you are showing a map to your users. If you hide the map and use these features of the plugin you are violating the Google Maps API Terms of Use!

  4. Usage Examples

    • You can have a single map located on your business or venue.

      You can set the map draggable property to false so the user won't be able to drag the marker.

      The map will just show off you place, but will allow the user to explore the surrounding area.

      It's perfect for events or small businesses.

    • You can also embed a static non-movable map which will act like an image to beautify your app.

      You even dynamically change the location to show a few places without interaction from the user.

    • You can have a form which prompts the user to point out a location.

      You can have a map with a movable marker which the user can easily drag and drop.

      And you can have a text field which will change the map dynamically with the user input.

      This way you will provide your users with very quick and easy way to point an exact location.

      And you will have exact coordinates plus a human readable addres which the user won't be entitled to write, but can customize for clarity.

    • You can ask for user's location with a map.

      So the user will be able to fix his location on the map if it isn't correct.

    • If you have a location-based site and you have a site search engine you cna show a map next to the results.

      For example if your user is searching for 'Buenos Aires' you can just set a map and use the setAddress method with the search term.

      The map will automatically be pointed to Buenos Aires, Argentina.

  5. Code Examples

    • Embed a map from Google Maps with just a single line of JavaScript

      If you want to just embed a Google map in you application just call the plugin on an empty div element.

      You can change the default options like default location, map type, zoom level and whether it should be draggable.

      To see how to change the full list of options see the all options demo source.

      <div id="embed-map" class="css-map"></div>

      <script>

          $('#embed-map').setmap();

      </script>

    • Adjust the map dimensions by pure CSS

      You can have any kind of map dimensions by just setting the width and the height of the div element with pure CSS.

      <div id="dimensions-map" style="width: 350px;height:200px;"></div>

      <script>

          $('#dimensions-map').setmap();

      </script>

    • Geocoding

      To geocode something means to take an address or a point and return its coordinates.

      <div id="geocoding-map" class="css-map"></div>

      <p>Move the marker to see geocoding in action.</p>

      <p class="lat">Latitude: <span></span></p>

      <p class="lng">Longitude: <span></span></p>

      <script>

          $('#geocoding-map').setmap({

              drop: function ( lat, lng ) {

                  var parent = $('#geocoding-map').parent();

                  $('.lat span', parent).text(lat);

                  $('.lng span', parent).text(lng);

              }

          });

      </script>

      The reversed geocoding is very useful when you need a human readable address of a place. Just give it a point and you will get the full address of it.

                  $('.address span', parent).text(address);

              }

    • HTML5 Geolocation

      HTML5 introduces a great way to get the current user location.

      When you set the current option as true the map center will default to the user's location.

      If the service fails or the user's browser doesn't support it, the center will fall back to the position object.

      <div id="geolocation-map" class="css-map"></div>

      <script>

          $('#geolocation-map').setmap({

              marker: {

                  current: true

              }

          });

      </script>

    • setAddress method

      You can set the location of the marker manually with the setAddress method.

      A great example is the demo with a map in a form.

      After you have initialized the plugin on a div element, just call it with 'setAddress' as a first parameter and the new address as a second.

      You can provide a callback to get the new point coordinates and a full address of the place.

      <div id="address-map" class="css-map"></div>

      <script>

          $('#address-map').setmap().setmap('setAddress', 'Tokyo, Japan', function ( lat, lng, address ) {

              var map = $('#address-map');

              map.after('<p>Latitude: ' + lat + '</p>');

              map.after('<p>Longitude ' + lng + '</p>');

              map.after('<p>Address ' + adress + '</p>');

          });

      </script>

    • Use the address components

      When you use reversed geocoding either with the drop callback or with the setAddress method you can use the address components.

      Just set the components parameter to true and these functions will return an array of objects as a third parameter.

      The breaking down of the address allows you to build more generic or more specific address for your application.

      <div id="components-map" class="css-map"></div>

      <script>

          $('#components-map').setmap({

              components: true,

              drop: function ( lat, lng, components) {

                  var container = $('#components-map').parent();

                  container.append('<p>Latitude: ' + lat + '</p>');

                  container.append('<p>Longitude ' + lng + '</p>');

                  $.each(components, function ( index, value ) {

                      container.append('<p>' + index + ': ' + value.long_name + '</p>');

                  });

              }

          });

      </script>

      Move the marker to see the new address broken into components.

    • Destroying a plugin instance

      When you don't want to use a setmap object anymore you better destroy it!

      When you call the destroy method the DOM element is reversed back to its previous state.

      <div id="destroy-map" class="css-map"></div>

      <p>

          <button id="map-destroy">Destroy</button>

      </p>

      <script>

          $('#destroy-map').setmap();

          $('#map-destroy').bind('click', function () {

              $('#destroy-map').setmap('destroy');

          });

      </script>

      Click the Destroy button to return the map div to its initial state.

  6. Options

    Here is a complete object for initializing a map with all the default values.

    If you don't specify a certain option the default will be used automatically.

    {
	map: {
		zoom: 12, // zoom level: from 0 to 20; higher is closer
		zoomControls: 'large', // 'small' or 'large' - defines whether a zoom slider will be present
		type: 'hybrid', // map type: hybrid, terrain, roadmap, satellite
		draggable: true, // should the users be able to drag the map
		disabled: false, // creates a static map with no visual, mouse or keyboard controls
		streetView: true // streetview controls
	},
	marker: {
		position: { // default location
			latitude: -34.397, 
			longitude: 150.644
		},
		current: false, // should the default location be the user's current location
		draggable: true, // should the users be able to drag the marker
		animation: 'drop', // 'drop' or 'bounce' - defines the marker animation
		title: '' // the marker title on hover
	},
	components: false, // global setting for address returned from reversed geocoding
	drop: function (latitude, longitude, fullAddress ) {
		// returns coordinates and full address after the marker is moved and dropped
	}
};

    Let's go through all of the available options:

    • map
      • zoom:

        The zoom property is a number between 0 and 20 and defines the default zoom level when tha map initializes.

        The user will be able to change the zoom level through visual, mouse and keyboard controls unless the disabled property is set to true.

      • zoomControls

        If set to true the map will have not only a +/- buttons for zooming, but also a zoom slider.

        If the disabled property is set to true the map will not have any zoom controls.

      • type

        The default map type for the map. There are four available values:

        • hybrid (default): a satellite map with main roads and other details
        • roadmap: detailed map of roads, buildings and others with no satellite imagery
        • satellite: a satellite map with only labels on top
        • terrain: a map showing the structure of the terrain, but not the details in the area

        The user will be able to change the map according to his/her preference unless the disabled property is set.

      • draggable

        If set to false the users will not be able to drag the map, but they will be able to zoom in and out, navigate with the keyboard and more.

        This option is useful when you want to show some area and the user be able to zoom in and out, but not to be able to go explore easily other areas.

      • disabled

        When set to true the created map is static and it has no zoom controls, no map types controls and no Street View.

        The user will not be able to drag the map or the marker and will no be able to navigate through the map or zoom in and out neither with the mouse nor the keyboard.

      • streetview

        If set to true the user will be able to drag the yellow Street View marker onto the map and observe the area and images of the area in Street View.

    • marker
      • position

        The position property is an object which defines the starting location of the marker. The map will be initially centered to this location as well.

        The position object contains latitude and longitude properties to define exact location.

        It is highly recommended to always provide the position option depending on the use case.

        For example if your business is located in Australia you will not want to initialize the map in the US and later navigate the user to your location.

        When the current property is set to true the position is not used.

      • current

        When set to true the initial location of the marker and map will be the current user's location.

        The plugin uses HTML5 Geolocation to define the location of the user.

        If the Geolocation fails, the user has forbidden the HTML5 Geolocation or the browser doesn't support it the map will fallback to the position property.

        So again it is highly recommended to set the position property to something useful.

        Note that if your user is using a private browsing option like Incognito in Google Chrome Geolocation will not work.

      • draggable

        If set to false the users will not be able to move the marker, but they will be able to drag around the map around and explore the area.

        This option is useful when you do not any location input from the user, but rather show something and its whereabouts on a map.

      • animation

        When the user drags and drops the marker a simple animation is shown.

        You can set it either to 'bounce' or 'drop'. The default is 'drop', because it is more cool.

      • title

        You can define a short text to be shown as a small tooltip next to the marker when the user hovers it.

    • components

      The components property is a global setting for the map related to the reversed geocoding functionality.

      When the components option is set to true the address from the reversed geocoding will be returned as an array of objects instead of a string.

      The default value is set to false, because in most cases you will want the full address.

      The address is built with every address component long name concated with a comma.

      If you want to use only some of the adress components (e.g. just city and state) then set this property to true.

      The reversed geocoding will return an array with object elements containing short and long name for every component.

      For more clarity see the components example.

    • drop

      The drop option is a handler function which is called every time the marker is dropped.

      It is called with three parameters - latitude, longitude and an address.

      So every time the user moves and drops the marker a geocoding and reversed geocoding is performed.

      This is very useful when you want a location input from your users to save the result in a database or something.

      Not only you get an exact location, but also a human readable address implementation of that location without the need to write it.

      Note that if haven't specified a function for the drop property the geocoding and the reversed geocoding on a marker drop is not performed and you won't loose performance of your application.

  7. Methods

    • setAddress

      This method allows you to change the map location dynamically with a partial address given.

      It performs a reversed geocoding and you get a latitude, a longitude and a full address.

      So you can use it not only to change the map location, but to get more location information for your partial address as well.

      If you have set the components options to true this method will have an array of components as a third parameter isntead of a string.

      The method is very useful to update a map from the user's input.

      You can check the map in a form demo for a great example.

    • setCoords

      This method is very similar to setAddress, but instead of a string addres you can set the map location through coordinates.

    • destroy

      The destroy method removes the map and restores the map DOM element to its previous state.

      If you work with lots of maps on the same page is essential to destroy your objects than rather just hiding them.

      After you have destroyed a setmap instance you can initalize it again on the same element with no problems. You can even use different options.
